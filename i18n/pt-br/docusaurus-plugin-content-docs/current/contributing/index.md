# Contribuindo

O Klar é experimental. Contribuições são bem-vindas, mas clareza é obrigatória.

Esta seção é focada em:
- como relatar issues corretamente
- como escrever documentação sem mentir sobre o estado do compilador

---

## Princípios para contribuidores

- Prefira explicitamente à conveniência
- Se um comportamento não estiver implementado, marque como **NI**
- Se um comportamento não estiver decidido, marque como **ND**
- Nunca “documente o futuro” como se ele já existisse hoje
