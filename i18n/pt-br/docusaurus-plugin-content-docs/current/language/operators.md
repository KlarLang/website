# Operadores

## Aritméticos

- `+ - * / %`

## Comparação

- `== != < > <= >=`

## Lógicos

- `and`
- `or`

`&&` e `||` não existem em Klar. Use `and/or`.

---

## Precedência

A precedência de operadores **ainda não está formalmente especificada** (ND).

Até que seja especificada, escreva Klar como se ambiguidade fosse um bug:
- use parênteses
- nunca confie em precedência implícita em expressões complexas
